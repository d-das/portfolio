"PlasTick" Gym App


Perhaps more of an obsession than a personal project, I have collaborated in building this app for the past year.

The project is an angular mobile app built within the ionic4 framework, with a simple backend based on firestore.
The goal of the project is to create a digital interface for bouldering gyms, where users can locate, discover, create, and comunicate about climbing routes.

The app can be broken down into several sections.


The Route Feed:
The feed can be viewed as a list, or as a map of the gym floor, both of which are populated by routes which can then be filtered.



The Route Details Page:
Detailed view of the route, with information about what kind of route it is, as well as send feedback and comment with other users about the route.


The Route Creator:
Here users create routes by taking a picture, drawing holds on the picture, locating the route on a map, and describing the route's features.


Analytics & Collections:
Users are then able to create custom lists or sets, as well as keep track of what routes they have climbed. In the future detailed analytics will be provided, so that users can understand their climbing and training from a deeper level.

Right now the project is in the last stages of development before a beta version will be released.

rofi, i3, & bash

Exploring the distance between my desktop enviroment and my thought processes.

Having been a linux user for almost a decade, I have tried many distributions and desktop enviroments. Eventually settling on a stripped-down version of ubuntu (mainly for stabilty), I always hated unity and gnome, finding them slow, gawky, and bloated. Eventually, I stumbled across i3, a simple tiling windows manager which changed the way I interact with my machine. It is minimal, customizable, does not waste any screen space, and incourages one to use a keyboard-driven work flow. That last part is crucial. Remanescent of vim, i3 is designed so that you never have to take your hands off the kayboard and fumble with a mouse. Therefore, a necessary aspect of this is the dmenu, an interface which allows you to open applications or run scripts. Enter rofi. Rofi is a "window switcher, application launcher and dmenu replacement". It is triggered when I press the keys "cmd+space".

Here is an example of an everyday use of rofi to open libre office:

I then noticed that I wanted to expand this interaction, not just to run local programs, but online ones too. IE, I wanted to google from my desktop.
So I wrote a simple bash script and then set the keyboard binding to something slightly different ("cmd+shift+enter"). Now, when rofi is opened, it basically takes whatever phrase I enter, and passes it on to firefox. This is akin to what alfred does for mac.
Here is the code:

#!/bin/bash

URL='https://www.google.com/?gws_rd=ssl#q='
QUERY=$(echo '' | rofi -dmenu -p "Google")
if [ -n "$QUERY" ]; then
 firefox -new-window "${URL}${QUERY}" 2› /dev/null
fi

This was probably the most useful thing I have written with the least amount of code. Still, I wnated to take it a bit further. More than just google, I frequently translate text from spanish to english and search things on wikipedia and wolfram. I wanted to expand the functionality to that of alfred, where I can type "g" for google, and "wi" for wiki, etc.
